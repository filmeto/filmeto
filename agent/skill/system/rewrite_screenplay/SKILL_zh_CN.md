# 剧本改写技能

本技能根据用户明确指示改写剧本场景。会读取现有内容，通过 LLM 按指示修改，并只写回改写后的剧本内容，从而保证指令被遵循且格式得以保留。

## 功能

- 根据清晰的用户指示改写单场或整部剧本
- 保持好莱坞剧本格式（场景标题、动作、角色名、对白）
- 保留场景元数据（地点、角色、一句话梗概等），除非指示中要求修改
- 确保输出严格遵循用户指示（语气、情节改动、对白修改等）

## 约束

- **需要上下文**：有效的 ToolContext，且具备剧本管理器和 LLM 服务
- **必须提供指示**：`instruction` 必填，描述要做的修改
- **范围**：使用 `scene_id` 仅改单场；不填或使用 "all" 则改写整部剧本
- **输出**：通过 LLM 只生成改写后的内容；保存的内容中不包含说明性文字

## 输入要求

通过 `execute_skill_script` 调用时：

- `instruction`（字符串，必填）：用户的改写指示（如「让对白更紧张」「把地点改成海滩」「把这场戏缩短一半」）。
- `scene_id`（字符串，可选）：若提供则只改写该场景；不填或为空则改写全部场景。
- `scope`（字符串，可选）：与 scene_id 二选一。设为 "all" 表示改写所有场景；未提供且未提供 scene_id 时默认为 "all"。

若缺少 `instruction`，技能将返回错误。不得自行推断指示内容，必须由用户明确说明要改什么。

## 使用说明

当用户以自然语言要求按描述修改剧本时调用（如「把开场改得更阴暗」「把 scene_002 的对白改得更自然」「把所有场景都改成夜间」）。技能会读取对应场景，将指示交给 LLM，并更新剧本，使结果符合指示。

**建议：**
- 传入清晰、具体的指示，便于模型严格遵循。
- 仅改单场时使用 `scene_id`；全局改动（语气、场景、风格）可不限定范围。

## 示例参数

```json
{
  "instruction": "让这场戏的对白更紧张，并增加两个角色之间的短暂冲突。",
  "scene_id": "scene_001"
}
```

```json
{
  "instruction": "缩短每一场戏，删去冗余动作描写，保留所有对白。",
  "scope": "all"
}
```

## 输出

返回 JSON：

- `success`: 布尔值
- `updated_scenes`: 被改写的 scene_id 列表
- `message`: 可读的摘要
- 出错时：`error`、`message` 说明原因

## 错误处理

- 缺少 `instruction` 会返回错误。
- 上下文中无剧本管理器或 LLM 会返回错误。
- 单场改写时未提供 scene_id 会返回错误。
- LLM 或更新失败会在响应中说明。
